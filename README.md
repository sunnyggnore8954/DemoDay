# 🪖 이미지 분석을 통해 헬멧 착용 여부를 판별하고, 공유 모빌리티의 안전한 이용을 지원하는 헬멘인식 AI 프로젝트

<br>
<br>

## 📌 프로젝트 한눈에 보기 :
2024년 SW동행 데모데이에서 공유 모빌리티 이용자의 헬멧 미착용 문제를 주제로 진행한 프로젝트입니다. 사용자가 입력한 이미지를 분석하여 헬멧 착용 시 SAFE, 미착용 시 DANGER로 판별하는 이미지 분류 모델을 개발했습니다. 

<br>

## 🚨 문제 정의 :
공유 전동킥보드와 같은 개인형 이동장치의 이용자가 증가하면서 관련 교통사고도 함께 늘어나고 있습니다. 그러나 실제 이용 과정에서는 헬멧 착용 의무가 제대로 지켜지지 않아, 사고 발생 시 머리와 얼굴에 심각한 부상을 입을 위험이 높습니다.
기존 제도는 헬멧 착용을 의무화하고 있지만, 사용자가 실제로 헬멧을 착용했는지 확인하기 어렵다는 한계가 있습니다. 이에 단순한 착용 권고를 넘어, 대여 전에 착용 여부를 자동으로 확인하는 방법을 구상했습니다. 

<br>

## 💡 해결 아이디어 :
사용자가 공유 모빌리티를 대여하기 전에 자신의 이미지를 입력하면, 인공지능이 헬멧 착용 여부를 판별하도록 설계했습니다.

ex) 헬멧 착용 이미지 → SAFE
헬멧 미착용 이미지 → DANGER
DANGER 판별 시 → 모빌리티 대여 제한 시스템과 연계

초기에는 CCTV를 이용해 미착용자를 실시간으로 탐지하는 방안도 검토했습니다. 하지만 설치 비용과 넓은 운영 범위를 고려했을 때, 공유 모빌리티 업체의 기존 대여 절차에 인공지능을 연결하는 방식이 더 실현 가능하다고 판단했습니다. 현재 프로젝트에서는 이미지 입력과 착용 여부 판별 기능까지 구현했으며, 실제 모빌리티 대여 제한 기능은 향후 연계 방향으로 제안했습니다. <br>

<br>


## 🎯 프로젝트 목표 :
이 프로젝트는 인공지능을 활용해 헬멧 미착용 문제를 줄이고, 공유 모빌리티 이용 과정의 안전성을 높이는 것을 목표로 했습니다.
(사용자의 헬멧 착용 여부를 이미지로 자동 판별 -> 모빌리티 이용 전 헬멧 착용을 자연스럽게 유도 -> 사고 발생 시 머리와 얼굴의 부상 위험 감소 -> 헬멧 착용 의무화 제도의 실효성 향상) <br>


<br>

## ⚙️ 개발 과정:
1. 문제 조사 및 서비스 기획
＞ 공유 모빌리티 사고 사례와 헬멧 미착용 현황을 조사하고, 인공지능을 어느 시점에 적용할지 논의했습니다. 이후 비용과 실현 가능성을 기준으로 모빌리티 대여 단계에서 착용 여부를 확인하는 흐름을 설계했습니다. 

2. 이미지 데이터 구성
＞ 헬멧을 착용한 이미지와 착용하지 않은 이미지를 수집하고, 모델이 각 상태를 구분할 수 있도록 라벨을 구성했습니다. 촬영 각도와 조명에 따라 인식 결과가 달라질 수 있어 다양한 환경의 이미지를 추가했습니다.

3. 모델 학습 및 평가
＞ 이미지의 형태와 특징을 학습할 수 있도록 CNN 기반 분류 모델을 구현했습니다. 학습용 데이터와 테스트용 데이터를 분리하여 모델 성능을 확인하고, 잘못 분류된 이미지를 분석해 데이터 구성을 보완했습니다.

4. 결과 시각화
＞ 학습 과정의 손실값과 평가 지표를 그래프로 확인하고, 혼동 행렬을 통해 실제 정답과 모델의 예측 결과를 비교했습니다. 새로운 이미지를 입력했을 때 SAFE 또는 DANGER 결과가 출력되는지도 테스트했습니다. <br>


<br>

## 🛠 사용 기술 :

- **TensorFlow / Keras** - CNN 이미지 분류 모델 설계와 학습
- **OpenCV** - 이미지 불러오기, 크기 조정 및 전처리

- **Scikit-learn** - 학습·테스트 데이터 분리 및 성능 평가
- **Matplotlib / Seaborn** - 학습 결과와 혼동 행렬 시각화
- **Google Colab** - 모델 학습 및 실행 환경 <br>

## 🌱 결과 및 배운 점 :
- 이미지 입력에 따라 헬멧 착용 여부를 SAFE와 DANGER로 구분하는 모델을 구현했습니다. 이 과정에서 인공지능 모델의 성능은 모델 구조뿐만 아니라 데이터의 품질, 촬영 각도, 조명, 라벨 정확도에 크게 영향을 받는다는 점을 확인했습니다.

- 인공지능을 단순한 이미지 분류 기능으로 끝내지 않고, 공유 모빌리티 대여 절차와 연결하여 실제 사회 문제를 해결하는 서비스로 확장하는 과정을 경험했습니다. 이를 통해 기술 구현뿐 아니라 문제 정의, 적용 방식, 실현 가능성을 함께 고려하는 개발 과정의 중요성을 배웠습니다.
